## üéôÔ∏è Creating Podcast Episodes

### Step-by-Step Guide

1. **Click "Podcast Episodes"** in the left sidebar
2. **Click "New Podcast Episode"** button (top right)
3. **Fill in the form** (see fields below)
4. **Click "Save"** (saves as draft) or **"Publish"** (goes live immediately)

### Required Fields

#### Basic Information
- **Episode Number**: Sequential number (1, 2, 3, etc.)
- **Title**: Episode title (e.g., "Understanding Autism Spectrum Disorder")
- **Slug**: URL-friendly version (auto-generated, but you can edit)
  - Example: `understanding-autism-spectrum-disorder`
  - Use lowercase letters and hyphens only

#### Guest Information
- **Guest Name**: Full name (e.g., "Dr. Temple Grandin")
- **Guest Bio**: 2-3 sentences about the guest
- **Guest Website**: Their website URL (optional)
- **Guest Photo**: Upload a photo (optional)

#### Episode Details
- **Topic/Category**: Choose from dropdown
  - Autism Awareness
  - Education
  - Social Development
  - Communication
  - Daily Living
  - Employment
  - Health & Wellness
  - Parenting
  - Personal Stories
  - Advocacy

- **Publish Date**: When this episode will go live
- **Duration**: Length in MM:SS format (e.g., "45:30")
- **Short Description**: Brief summary (150-200 characters)
  - This appears in the episode list
  - Keep it concise and engaging

#### Audio & Content
- **Audio File**: Click "Choose file" and upload your MP3
  - Recommended: MP3 format, 128kbps or higher
  - File size: Under 100MB

- **Show Notes**: Detailed notes about the episode
  - Use the rich text editor
  - Include timestamps (e.g., "00:05:30 - Topic discussion")
  - Add headings, bullet points, and formatting

#### Resources
- **Resources Mentioned**: Click "+ Add Resource" for each link
  - Resource Title: Name of the resource
  - Resource URL: Web address
  - Description: Brief note about the resource (optional)

#### Metadata
- **Featured Episode**: Check this to show on homepage
- **Season**: Season number (optional)
- **Transcript Available**: Check if you have a transcript
- **Transcript**: Paste the full transcript (optional)

### Example: Creating Episode 1

```
Episode Number: 1
Title: Understanding Autism Spectrum Disorder
Slug: understanding-autism-spectrum-disorder
Guest Name: Dr. Temple Grandin
Guest Bio: Dr. Temple Grandin is a professor of animal science...
Topic: Autism Awareness
Publish Date: 2024-01-15
Duration: 45:30
Description: Join us for an insightful conversation...
Audio File: [Upload episode-001.mp3]
Show Notes: [Write detailed notes with timestamps]
Resources:
  - Title: Temple Grandin Official Website
    URL: https://www.templegrandin.com/
Featured: ‚úì (checked)
```

## üìù Creating Blog Posts

### Step-by-Step Guide

1. **Click "Blog Posts"** in the left sidebar
2. **Click "New Blog Post"** button
3. **Fill in the form**
4. **Click "Save"** or **"Publish"**

### Required Fields

#### Basic Information
- **Title**: Blog post title
  - Make it clear and descriptive
  - Example: "10 Essential Tips for Navigating Autism Diagnosis"

- **Slug**: URL-friendly version (auto-generated)
  - Example: `navigating-autism-diagnosis-tips`

- **Author**: Select from dropdown
  - ARP Team
  - Dr. Sarah Johnson
  - Emily Rodriguez
  - Michael Chen
  - Guest Contributor

- **Publish Date**: When this post will go live

#### Categorization
- **Category**: Choose one main category
  - Getting Started
  - Education
  - Communication
  - Daily Living
  - Parenting
  - Resources
  - Personal Stories
  - Health & Wellness
  - Advocacy
  - Research

- **Tags**: Add keywords (comma-separated)
  - Example: diagnosis, early intervention, support, parents
  - These help with search

#### Images & Summary
- **Excerpt**: Short summary (150-200 characters)
  - This appears in the blog list
  - Make it compelling!

- **Featured Image**: Upload main image
  - Recommended size: 1200x630 pixels
  - JPG or PNG format

- **Image Alt Text**: Describe the image
  - Important for accessibility
  - Example: "Parent holding child's hand"

- **Read Time**: Estimated reading time
  - Example: "8 min read"
  - You can estimate or use a word count tool

#### Content
- **Content**: Write your blog post
  - Use the rich text editor
  - Add headings (H2, H3) to organize
  - Include bullet points and numbered lists
  - Add images inline if needed
  - Format text (bold, italic, links)

#### SEO Settings (Optional)
- **SEO Title**: Custom title for search engines
  - Defaults to your post title if left blank
  
- **SEO Description**: Custom description
  - Defaults to your excerpt if left blank

- **Keywords**: Additional keywords for SEO

#### Metadata
- **Featured Post**: Check to show on homepage
- **Allow Comments**: Check to enable comments
- **Related Posts**: Add slugs of related posts

### Writing Tips

1. **Start with a strong opening** - Hook readers in the first paragraph
2. **Use clear headings** - Break up content with H2 and H3 headings
3. **Keep paragraphs short** - 2-4 sentences per paragraph
4. **Use bullet points** - Make lists easy to scan
5. **Add examples** - Real-world examples help readers understand
6. **Include calls-to-action** - Link to other resources
7. **Proofread** - Check spelling and grammar before publishing

## üìã Editorial Workflow

### Understanding the Workflow

Content goes through three stages:

1. **Drafts** üìù
   - Work in progress
   - Only visible to you
   - Can be edited freely

2. **In Review** üëÄ
   - Ready for review
   - Waiting for approval
   - Can still be edited

3. **Ready** ‚úÖ
   - Approved and ready to publish
   - Will go live on publish date

### How to Use the Workflow

#### Creating a Draft
1. Create your content
2. Click **"Save"** (not "Publish")
3. Content is saved as a draft

#### Submitting for Review
1. Open your draft
2. Click **"Set status"** ‚Üí **"In Review"**
3. Reviewers will be notified

#### Publishing Content
1. Content must be in "Ready" status
2. Click **"Publish"**
3. Content goes live immediately (or on scheduled date)

### Collaboration Tips

- **Save often** - Click "Save" frequently to avoid losing work
- **Use clear titles** - Make it easy for reviewers to identify content
- **Add notes** - Use the notes field to communicate with reviewers
- **Check before publishing** - Always preview before publishing

## üí° Tips & Best Practices

### General Tips

1. **Save your work frequently** - Click "Save" every few minutes
2. **Use the preview** - Check how content looks before publishing
3. **Fill in all required fields** - Red asterisks (*) indicate required
4. **Use descriptive titles** - Make content easy to find
5. **Add alt text to images** - Important for accessibility
6. **Proofread** - Check spelling and grammar
7. **Test links** - Make sure all URLs work
8. **Use consistent formatting** - Follow the same style across content

### Image Guidelines

- **File size**: Keep under 2MB for fast loading
- **Dimensions**: 
  - Blog featured images: 1200x630px
  - Thumbnails: 400x300px
  - Guest photos: 400x400px
- **Format**: JPG for photos, PNG for graphics
- **Alt text**: Always add descriptive alt text

### Writing Style

- **Be clear and concise** - Use simple language
- **Use active voice** - "We provide" not "Services are provided"
- **Break up text** - Use headings, lists, and short paragraphs
- **Be inclusive** - Use person-first or identity-first language as appropriate
- **Cite sources** - Link to original sources when referencing research

### SEO Best Practices

- **Use keywords naturally** - Don't stuff keywords
- **Write compelling titles** - Include main keyword
- **Create good meta descriptions** - Summarize content in 150-160 characters
- **Use headings** - Structure content with H2 and H3 tags
- **Link internally** - Link to other relevant content on the site
- **Add alt text** - Helps with image search

## üîß Troubleshooting

### Common Issues

#### "I can't log in"
- **Solution**: Make sure you have a GitHub account and have been added to the repository
- **Contact**: Ask the site administrator to add you

#### "My changes aren't showing up"
- **Solution**: 
  1. Make sure you clicked "Publish" (not just "Save")
  2. Wait 2-3 minutes for deployment
  3. Clear your browser cache
  4. Try a different browser or incognito mode

#### "I accidentally deleted something"
- **Solution**: 
  1. Don't panic! Content is version controlled
  2. Contact the site administrator
  3. They can restore from GitHub history

#### "The rich text editor isn't working"
- **Solution**:
  1. Try refreshing the page
  2. Clear browser cache
  3. Try a different browser
  4. Make sure JavaScript is enabled

#### "I can't upload files"
- **Solution**:
  1. Check file size (must be under 10MB)
  2. Check file format (PDF, JPG, PNG, MP3)
  3. Try a different browser
  4. Contact administrator if issue persists

#### "My content is stuck in 'In Review'"
- **Solution**:
  1. Contact a reviewer or administrator
  2. They need to approve it before it can be published
